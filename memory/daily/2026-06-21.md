## 新增：修复 hydration mismatch 与全站 smoke test（同日晚间）

### 问题

用户反馈 `/archetypes` 页面出现 React hydration 错误：

```
Hydration failed because the server rendered HTML didn't match the client.
```

根因：多个 Client Component 在渲染阶段直接使用 `typeof document !== "undefined"` / `typeof window !== "undefined"` 分支，导致服务端与客户端首次渲染结果不一致。

### 已修复的组件

| 文件 | 问题 | 修复方式 |
|---|---|---|
| `src/components/demos/DemoEmbed.tsx` | 全屏按钮基于 `document.fullscreenEnabled` 条件渲染 | 使用 `useCanFullscreen()` |
| `src/components/demos/FullscreenStage.tsx` | 同上 | 使用 `useCanFullscreen()` |
| `src/components/demos/GameShell.tsx` | 同上，且 hook 被条件调用 | 使用 `useCanFullscreen()`，新增 `showFullscreenButton` |
| `src/components/site/TopNav.tsx` | 搜索框初始值读取 `window.location.search` | 改为非受控输入，`defaultValue` 通过 `useClientValue` 获取 |
| `src/components/cookie/CookieConsent.tsx` | 横幅可见性在 effect 中 setState | 使用 `useClientValue` + `dismissed` 状态 |
| `src/components/analytics/GoogleAnalytics.tsx` | GA 加载状态在 effect 中 setState | 使用 `useClientValue` |

### 新增 Hook

- `src/lib/hooks/useClientValue.ts`：基于 `useSyncExternalStore`，hydration 期间使用服务端快照，完成后切换到客户端值。
- `src/lib/hooks/useCanFullscreen.ts`：基于 `useClientValue` 的全屏能力检测。

### 新增全站 smoke test

- 脚本：`scripts/smoke-test.ts`
- 依赖：`puppeteer-core`（使用系统 Chrome，无需下载浏览器）
- 命令：`pnpm test:smoke`
- 覆盖：68 个页面（首页、合规页、母型/原型/特征列表与详情、30 个玩法详情、版主后台、Demo 页）

  - HTTP 200

  - console.error 中无 hydration 错误
  - 页面存在 `<main>` 或 `<h1>`
  - 无 React Runtime / Hydration 错误遮罩
