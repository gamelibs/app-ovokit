# OVOFORGE 当前阶段目标

> 阶段：内部 MVP 运行与架构扩展
> 时间范围：2026-06 起

---

## 总体目标

把站点从"12 种母型玩法目录"扩展为 **"12 母型 + 5 种核心编辑器原型"的双层分类体系**，让：

- **用户**可以按"母型感受"或"编辑器架构"两种维度浏览玩法
- **版主**发帖时能选择核心原型，并获得对应的内容骨架与 Demo 模板
- **Demo 体系**能按核心原型模板化生成，降低新增玩法的 Demo 成本

---

## 里程碑

### 里程碑 1：核心原型分类层跑通
**验收标准**：
- 5 种核心原型有稳定的数据定义（`src/lib/patterns/patterns.ts`）
- `PlayMeta` 支持 `pattern` 字段
- 首页筛选新增"核心原型"分组，能按原型过滤帖子
- 版主发帖表单可选择核心原型
- API 保存/读取透传 `pattern`

### 里程碑 2：母型与核心原型映射展示
**验收标准**：
- 12 种母型各自映射到 1~N 种核心原型
- 母型详情页展示所属核心原型及其抽象循环
- 核心原型页面（可选）展示属于该原型的母型和玩法

### 里程碑 3：5 种核心原型最小 Demo
**验收标准**：
- 每个核心原型有一个服务端 Demo（`server/demos/patterns/<key>/`）
- 每个核心原型有嵌入页面（`/embed/demos/pattern/<key>`）
- 详情页 Demo 回退逻辑优先使用帖子的 `pattern`，其次母型
- 版主选择原型后，表单能推荐默认 Demo iframe 地址

### 里程碑 4：版主工作流模板化
**验收标准**：
- 选择核心原型后，自动生成对应 `breakdown` 章节骨架
- 自动生成对应 `codeSnippets` 最小代码模板
- AI 分析工具能把游戏识别到核心原型，并写入 `pattern` 字段

---

## 非目标（本阶段不做）

- 不替换现有 12 母型体系
- 不改站点视觉风格（手绘风）
- 不引入用户登录/评论等社交功能
- 不迁移统计持久化（文件系统 views 在内部单实例运行可接受）
- 不补齐真实封面图/母型图（继续用占位图）
