# OVO 版主工作流

> 本文档描述版主（Moderator）从 AI 分析游戏到发布 Play 帖子的完整工作流程。

---

## 一、版主入口

1. 在网站首页连续点击顶部 **「OVO」Logo 8 次**
2. 输入版主口令（`MOD_PASSWORD`，配置在 `.env.local`）
3. 登录成功后，版主 Cookie 写入浏览器
4. 自动跳转到 **内容管理**（`/mod`）

---

## 二、内容管理（/mod）

版主登录后的默认首页。功能：

- 查看所有 Play 帖子列表（ID、标题、状态、发布时间、浏览量、点赞数）
- 排序：按发布时间 / 浏览量 / 点赞数（点击表头切换升序/降序）
- 分页：每页 10 条
- 操作：
  - **发布/下架**：切换帖子的 published 状态
  - **编辑**：进入编辑页面（`/mod/edit/<slug>`）
  - **删除**：永久删除帖子及其文件
- **新建玩法**：跳转到发帖表单（`/mod/new`）

---

## 三、AI 辅助分析（/mod/tools）

点击内容管理页面的 **"开发者工具箱"** 或直接从 `/mod/tools` 进入。

### 3.1 四大 AI 工具

| 模式 | 输入 | 输出 | 用途 |
|------|------|------|------|
| 🔬 **全面分析** | 截图(可选) + URL(可选) + 源码目录(可选) | 完整的结构化帖子数据 | 一键生成帖子草稿 |
| 🎨 **生成封面图** | 截图(必需) + 标题(可选) | SVG 封面（5种风格可选） | 帖子封面素材 |
| 📊 **生成流程图** | 源码目录(必需) + 截图(可选) | Mermaid 流程图 | 文章插图/逻辑梳理 |
| ✏️ **生成示意图** | 截图(必需) | 界面 wireframe SVG（4种风格） | 文章插图 |

### 3.2 封面图风格

- **手绘风格**（默认）：不规则抖动线条，温暖柔和配色
- **扁平插画**：简洁几何形状，纯色填充
- **像素艺术**：8-bit/16-bit 复古像素网格
- **霓虹发光**：深色背景 + 霓虹色系 + drop-shadow
- **极简线条**：单色细线勾勒，大面积留白

### 3.3 示意图风格

- **手绘线框**（默认）：线条有抖动感，纸张手绘质感
- **纯线框图**：单色线条，无填充
- **彩色填充**：不同功能区用浅色填充区分
- **等距 3D**：30度角等距投影，三维结构感

### 3.4 全面分析 → 创建帖子 工作流

```
1. 进入 /mod/tools → 点击"🤖 分析游戏"
2. 选择"🔬 全面分析"模式（默认）
3. 上传游戏截图（推荐）+ 填写游戏地址（可选）+ 填写源码目录（可选）
4. 点击"开始全面分析"
5. 进度遮罩显示阶段：
   "正在收集分析素材..."
   → "AI 正在识别游戏类型和机制..."
   → "AI 正在提炼核心玩法逻辑..."
   → "AI 正在整理技术实现要点..."
   → "AI 正在生成结构化报告..."
6. 结果页展示所有字段（可编辑）：
   ├─ 基础信息：标题/副标题/Slug/难度/标签/技术栈/核心点
   ├─ 玩法拆解：N 个章节预览
   ├─ 代码片段：N 个片段预览
   ├─ 流程图：Mermaid 代码预览
   └─ 文章：MDX 全文（可编辑）
7. 点击"📝 创建新帖"
   → 数据写入 localStorage (ovofroge:analyzer-draft)
   → 自动跳转 /mod/new
8. NewPlayForm 检测到 analyzer-draft，自动填充所有字段
9. 版主补充封面（可用 AI 封面工具生成）、Demo iframe、横向封面
10. 点击"发布玩法"
```

### 3.5 AI 映射表

| Play 字段 | AI 来源 | 生成方式 |
|-----------|---------|----------|
| title, subtitle, slug | `analyze-game` | AI 直接生成 |
| tags | `analyze-game` | AI 从预定义标签中选择 2-4 个 |
| difficulty | `analyze-game` | AI 判断（入门/进阶/硬核）|
| techStack | `analyze-game` | AI 推断技术栈 |
| corePoints | `analyze-game` | AI 提炼核心设计点 |
| breakdown | `analyze-game` | AI 生成章节 + 要点列表 |
| codeSnippets | `analyze-game` | AI 从源码提取关键代码 |
| articleMdx | `analyze-game` | AI 生成完整 MDX 文章 |
| cover | `generate-cover` | AI 生成 SVG 封面（5种风格）|
| flowchart | `generate-flowchart` | AI 生成 Mermaid 流程图 |
| sketch | `generate-sketch` | AI 生成 wireframe SVG（4种风格）|

## 四、新建玩法（/mod/new）

从内容管理点击 **"新建玩法"** 进入。

### 4.1 表单字段

**基础信息**：
- 标题（必填）
- 副标题（必填）
- Slug（必填，用于 URL）
- 难度：入门 / 进阶 / 硬核
- 标签（多选，从预定义标签中选择）
- 技术栈（多选）
- 核心点（多选）

**封面**：
- 竖向封面 3:4（必需，支持 png/jpg/webp/svg，最大 5MB）
- 横向封面 4:3（可选）
- AI 手绘封面生成器（内置 Rough.js 模板）

**玩法拆解**：
- 章节式编辑器，每章包含标题 + 要点列表
- 支持拖拽排序

**代码片段**：
- 多片段编辑器
- 支持语言：TypeScript / TSX / JavaScript / GLSL / JSON / MDX
- 支持拖拽排序

**Demo**：
- iframe 地址（可选）
- 演示视频上传（mp4/webm，最大 50MB）
- Demo 说明文字

**文章（MDX）**：
- Markdown + JSX 编辑器
- 支持 AI 一键生成文章（基于表单数据）

### 4.2 自动草稿

- 表单自动保存到 `localStorage`（每 400ms 防抖）
- 刷新页面后自动恢复
- 支持手动清除草稿

### 4.3 从 AI 分析自动填充

当从 GameAnalyzer "全面分析"模式点击"创建新帖"时：
- 所有 AI 生成的字段自动填充到表单
- 封面需版主手动上传或生成
- Demo 需版主手动配置

## 五、编辑玩法（/mod/edit/<slug>）

从内容管理点击 **"编辑"** 进入。

- 加载已有的 `meta.json` + `article.mdx`
- 表单预填充所有数据
- **不允许修改 Slug**（避免资源与链接错位）
- 支持覆盖写入
- 支持重新上传封面/视频

---

## 六、案例演示（/mod/cases）

展示所有已配置 `demo.iframeSrc` 的帖子，便于：
- 快速查看哪些帖子有 Demo
- 复制 Demo 链接/嵌入地址
- 跳转到对应帖子编辑页

**挂载 Demo 到帖子的流程**：
1. 在 `/demo/blocks` 进入交互编辑器，搭建 Demo
2. 复制页面链接或 iframe 地址
3. 到内容管理编辑对应玩法，在 `demo.iframeSrc` 填入链接
4. 保存后，访客在帖子详情页即可交互体验


## 八、内容存储结构

```
content/plays/<slug>/
  ├── meta.json      # 结构化元数据
  └── article.mdx    # 文章正文

public/plays/<slug>/
  ├── cover.*        # 竖向封面
  ├── cover-wide.*   # 横向封面（可选）
  └── demo.*         # Demo 视频（可选）
```

## 九、必要素材清单

版主发布一个完整的 Play 帖子，至少需要：

| # | 素材 | 来源 | 说明 |
|---|------|------|------|
| 1 | 标题 + 副标题 | AI / 手动 | 一句话概括玩法核心 |
| 2 | 标签 + 难度 | AI / 手动 | 用于分类和筛选 |
| 3 | 封面图 | AI / 手动 | 竖向 3:4，首页展示 |
| 4 | 玩法拆解 | AI / 手动 | 至少 2 个章节 |
| 5 | 文章正文 | AI / 手动 | MDX 格式 |

推荐补充：
- 技术栈、核心点
- 代码片段
- 流程图（Mermaid）
- 界面示意图（wireframe SVG）
- Demo（iframe 或可玩链接）
- 横向封面

---

## 十、技术依赖

- **AI 引擎**：Kimi K2.6（通过 Anthropic-compatible API）
- **手绘 SVG**：Rough.js（服务端生成）
- **SVG → 位图**：Canvas API（客户端导出 PNG/JPG/WebP）
- **内容存储**：文件系统（`content/plays/` + `public/plays/`）
- **版主鉴权**：Cookie + 环境变量密码
