# OVOKIT 手绘风格契约（Style Contract）

> 目标：通过代码约束 + 设计令牌（Design Tokens）+ 验收规则，确保任何贡献者（人 / AI Agent）产出的 UI 都严格落在统一的手绘风格内，不出现偏移。

---

## 一、核心原则：代码即设计稿

手绘风格的稳定性不依赖"人眼审美"，而依赖**不可覆盖的代码约束**：

1. **颜色不由开发者随意写死** → 只能从 CSS 变量取用
2. **边框不由开发者随意写** → 统一使用 `<SketchBorder>` 组件
3. **卡片不由开发者随意写** → 统一使用 `<SketchCard>` 组件
4. **图标不由开发者随意写** → 统一使用 `<SketchIcon>` 组件
5. **字体不由开发者随意写** → Tailwind 配置锁定

> **规则：任何偏离契约的 PR，lint 自动报错。**

---

## 二、设计令牌（Design Tokens）

所有视觉值提取为 CSS 变量，全局唯一真源。

```css
/* globals.css —— 手绘风格变量区块 */
:root {
  /* 纸张背景 */
  --paper-bg: #faf7ef;
  --paper-bg-warm: #f5f1e6;
  --paper-grid: #e8e0d0;        /* 可选：浅色网格线 */

  /* 墨水色 */
  --ink: #202020;
  --ink-light: #555555;
  --ink-muted: #888888;

  /* 强调色（手绘马克笔感） */
  --highlight-yellow: #ffda6a;
  --highlight-blue: #7dcfff;
  --highlight-red: #ff8b8b;
  --highlight-green: #7dd87d;
  --highlight-orange: #ffb366;

  /* RoughJS 参数 */
  --sketch-roughness: 2;
  --sketch-bowing: 1;
  --sketch-stroke-width: 2;
  --sketch-stroke: var(--ink);

  /* 组件参数 */
  --sketch-radius: 12px;          /* 手绘圆角 */
  --sketch-card-rotate-min: -1deg;
  --sketch-card-rotate-max: 1deg;
  --sketch-card-shadow: 2px 3px 0px rgba(32, 32, 32, 0.15);
}
```

> **约束**：任何组件的 `background`、`color`、`border-color` 必须从上述变量取值，禁止写死 `#faf7ef`、`#202020` 等字面量。

## 三、字体加载契约

### 加载方式

通过 `next/font/google` 加载，确保性能与稳定性：

```typescript
// src/lib/fonts.ts
import { Inter, Kalam } from 'next/font/google';

export const inter = Inter({
  subsets: ['latin'],
  variable: '--font-inter',
  display: 'swap',
});

export const kalam = Kalam({
  weight: ['400', '700'],
  subsets: ['latin'],
  variable: '--font-kalam',
  display: 'swap',
});
```

### 使用规则

| 元素 | 字体 | Tailwind 类 |
|------|------|-------------|
| 页面标题 H1 | Kalam | `font-kalam` |
| 模块标题 H2/H3 | Kalam | `font-kalam` |
| 卡片标题 | Kalam | `font-kalam` |
| 正文/段落 | Inter | `font-sans` |
| 代码/标签 | Inter | `font-mono` |

> **约束**：禁止在组件内直接使用 `font-family: 'xxx'`；必须通过 Tailwind 类或 CSS 变量。

## 四、组件级强制规范

### 4.1 SketchBorder（手绘边框）

所有带边框的元素**必须**通过此组件或此 Hook 实现，禁止手写 `border: 2px solid #222`。

```typescript
// src/components/sketch/SketchBorder.tsx
'use client';

import { useRef, useEffect } from 'react';
import rough from 'roughjs';

interface SketchBorderProps {
  children: React.ReactNode;
  className?: string;
  roughness?: number;
  bowing?: number;
  stroke?: string;
  strokeWidth?: number;
  fill?: string;
  radius?: number;
}

export function SketchBorder({
  children,
  className = '',
  roughness = 2,
  bowing = 1,
  stroke = '#202020',
  strokeWidth = 2,
  fill = 'none',
  radius = 12,
}: SketchBorderProps) {
  const svgRef = useRef<SVGSVGElement>(null);
  const containerRef = useRef<HTMLDivElement>(null);

  useEffect(() => {
    if (!svgRef.current || !containerRef.current) return;
    const { width, height } = containerRef.current.getBoundingClientRect();
    const svg = svgRef.current;
    svg.setAttribute('width', String(width));
    svg.setAttribute('height', String(height));
    svg.innerHTML = '';

    const rc = rough.svg(svg);
    const node = rc.rectangle(
      strokeWidth / 2,
      strokeWidth / 2,
      width - strokeWidth,
      height - strokeWidth,
      {
        roughness,
        bowing,
        stroke,
        strokeWidth,
        fill,
        fillStyle: 'hachure',
      }
    );
    svg.appendChild(node);
  }, [roughness, bowing, stroke, strokeWidth, fill]);

  return (
    <div ref={containerRef} className={`relative ${className}`}>
      <svg
        ref={svgRef}
        className="pointer-events-none absolute inset-0"
        style={{ width: '100%', height: '100%' }}
      />
      <div className="relative z-10">{children}</div>
    </div>
  );
}
```

### 4.2 SketchCard（手绘卡片）

所有卡片**必须**使用此组件。

```typescript
// src/components/sketch/SketchCard.tsx
'use client';

import { useMemo } from 'react';
import { SketchBorder } from './SketchBorder';

interface SketchCardProps {
  children: React.ReactNode;
  className?: string;
  rotate?: 'random' | number;   // 'random' 使用预设范围，number 为固定角度
}

export function SketchCard({
  children,
  className = '',
  rotate = 'random',
}: SketchCardProps) {
  const angle = useMemo(() => {
    if (typeof rotate === 'number') return rotate;
    // 伪随机但稳定：基于内容 hash 或固定种子
    // 简单实现：固定 3 个档位
    const angles = [-0.8, 0, 0.8];
    return angles[Math.floor(Math.random() * angles.length)];
  }, [rotate]);

  return (
    <div
      className={`transition-transform duration-200 hover:scale-[1.01] ${className}`}
      style={{ transform: `rotate(${angle}deg)` }}
    >
      <SketchBorder fill="rgba(250, 247, 239, 0.8)">
        <div className="p-4">{children}</div>
      </SketchBorder>
    </div>
  );
}
```

> **约束**：
> - 卡片旋转角度限制在 `[-1deg, +1deg]`
> - 禁止超过此范围，否则破坏"微手绘感"
> - 禁止 3D transform、禁止 box-shadow（用 SVG 描边模拟阴影）

### 4.3 SketchButton（手绘按钮）

```typescript
// src/components/sketch/SketchButton.tsx
'use client';

import { SketchBorder } from './SketchBorder';

interface SketchButtonProps {
  children: React.ReactNode;
  onClick?: () => void;
  variant?: 'primary' | 'secondary' | 'ghost';
  className?: string;
}

export function SketchButton({
  children,
  onClick,
  variant = 'primary',
  className = '',
}: SketchButtonProps) {
  const fillMap = {
    primary: 'var(--highlight-yellow)',
    secondary: 'var(--paper-bg)',
    ghost: 'none',
  };

  return (
    <button onClick={onClick} className={`cursor-pointer ${className}`}>
      <SketchBorder fill={fillMap[variant]} radius={12}>
        <span className="font-kalam px-4 py-2 block">{children}</span>
      </SketchBorder>
    </button>
  );
}
```

### 4.4 SketchDivider（手绘分隔线）

```typescript
// src/components/sketch/SketchDivider.tsx
'use client';

import { useRef, useEffect } from 'react';
import rough from 'roughjs';

export function SketchDivider({ className = '' }: { className?: string }) {
  const svgRef = useRef<SVGSVGElement>(null);

  useEffect(() => {
    if (!svgRef.current) return;
    const svg = svgRef.current;
    const rc = rough.svg(svg);
    const node = rc.line(0, 10, 200, 10, {
      roughness: 2,
      bowing: 1,
      stroke: '#202020',
      strokeWidth: 2,
    });
    svg.appendChild(node);
  }, []);

  return (
    <div className={`w-full overflow-hidden ${className}`}>
      <svg ref={svgRef} width="100%" height="20" viewBox="0 0 200 20" preserveAspectRatio="none" />
    </div>
  );
}
```

### 4.5 SketchIcon（手绘图标）

所有图标通过 Lucide，统一参数：

```typescript
// src/components/sketch/SketchIcon.tsx
import { icons, type LucideIcon } from 'lucide-react';

interface SketchIconProps {
  name: keyof typeof icons;
  size?: number;
  className?: string;
}

export function SketchIcon({ name, size = 24, className = '' }: SketchIconProps) {
  const Icon = icons[name];
  if (!Icon) return null;
  return <Icon size={size} strokeWidth={2} className={`text-ink ${className}`} />;
}
```

> **约束**：禁止使用 3D 图标、彩色图标、拟物图标。所有图标必须 `strokeWidth={2}`。

## 七、SVG 素材生成契约

### 7.1 输入规范

任何 AI 或人工生成的 SVG，必须遵循：

```json
{
  "title": "玩法名称",
  "type": "母型分类",
  "core": "核心机制一句话",
  "style": "hand-drawn",
  "strokeWidth": 2,
  "roughness": 2,
  "palette": ["#202020", "#ffda6a", "#7dcfff"]
}
```

### 7.2 输出规范

生成的 SVG 必须：
- 纯 SVG，无外部图片引用
- `viewBox` 明确
- 所有路径使用 `stroke-linecap="round"`、`stroke-linejoin="round"`
- 无 `style` 内联属性（使用 `stroke`/`fill` 属性）
- 经过 SVGO 压缩后入库

### 7.3 目录规范

```
public/svg/
  icons/           # 站点通用图标
  illustrations/   # 玩法相关插图
  flows/           # 流程图
  covers/          # 封面底图模板
```

## 八、验收清单（防止偏移）

任何页面改造完成后，必须通过以下检查：

- [ ] 页面背景色是 `bg-paper`（`#faf7ef`），不是纯白或其他色
- [ ] 标题使用 `font-kalam`，正文使用 `font-sans`
- [ ] 所有卡片使用 `<SketchCard>`，有微旋转效果
- [ ] 所有按钮使用 `<SketchButton>`，有手绘边框
- [ ] 所有图标使用 `<SketchIcon>`，`strokeWidth={2}`
- [ ] 无手写 `border` / `box-shadow` / `background-color` 字面量
- [ ] 无 PNG/JPG/WebP 装饰性素材（封面/图标/分隔线全部 SVG）
- [ ] 移动端触摸无异常（RoughJS SVG 不干扰 `touch-action`）
- [ ] Dark Mode 已决定保留或废弃（见下方决策）

## 九、待决策项

1. **Dark Mode 策略**
   - A. 废弃 Dark Mode（手绘风格在深色下难协调）
   - B. 保留 Dark Mode，但手绘风格在 Dark 下使用深色纸张 + 浅色墨水

2. **卡片旋转策略**
   - A. 完全随机（每次刷新角度不同）
   - B. 伪随机但稳定（基于 slug hash，同一页面永远相同角度）

3. **RoughJS 渲染性能**
   - 大量卡片同时渲染 RoughJS SVG 可能影响性能，是否需要：
   - A. 仅在首屏关键卡片使用 RoughJS，其他用 CSS 模拟
   - B. 全部使用 RoughJS，接受性能代价
   - C. 服务端预渲染 SVG 路径（build 时生成静态 SVG）
