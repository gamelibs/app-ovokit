## 一、执行摘要

OVOFROGE V1 是一个面向游戏开发者的**手绘风格玩法知识分享站**。核心功能已完整实现：用户可浏览、搜索、阅读玩法拆解，版主可发布和管理内容。

**当前状态**：技术骨架 100% 完成，手绘风格 100% 覆盖，但存在**数据真实化缺失**和**内容占位符**两个主要缺口。

**V1 生产级的定义**：
> 对外公开访问时，用户看不到假数据、看不到占位符、不会遇到死链，版主能正常发帖管理，站点具备基础 SEO 和互动能力。

**最高优先级任务（4项）**：
1. 首页数据真实化（热门玩法/母型数量从硬编码 → 动态计算）
2. 详情页相关推荐（同母型/同标签玩法推荐）
3. 内容占位符清理（article.mdx 中的 "MVP 占位" 文本）
4. 联系表单实际可用（从静态展示 → 可提交）

## 二、V1 产品定位（当前已实现）

### 2.1 一句话定位

OVOFROGE 是一个**手绘风格的结构化玩法知识库**，把每个游戏玩法拆解为：规则说明 → 关键代码 → 可试玩 Demo → 设计文章。

### 2.2 内容模型（当前已实现）

```
├─ 玩法帖子（Play）
│   ├─ 标题 / 副标题 / 标签 / 难度 / 技术栈
│   ├─ 玩法拆解（结构化 JSON）
│   ├─ 关键代码片段
│   ├─ Demo（iframe / video / 母型回退）
│   └─ 文章（MDX）
│
├─ 母型玩法（Archetype）
│   └─ 12 个固定母型，作为分类锚点
│
└─ 嵌入式演示（Embed）
    └─ iframe 隔离，避免滚动/交互冲突
```

## 四、V1 生产级标准

### 4.1 什么是"生产级"

对外公开时，用户视角的验收标准：

1. **看不到假数据** — 首页热门、母型数量、推荐内容都是基于真实内容计算
2. **看不到占位符** — 没有 "MVP 占位"、"TODO"、"后续补充" 等文本

4. **能正常互动** — 搜索、点赞、浏览计数正常工作
5. **版主能管理** — 发帖、编辑、删除、发布状态控制完整可用

## 六、最高优先级实施任务

### 任务 1：首页数据真实化（20min）

**问题**：`HotPlaysSection.tsx` 硬编码 5 个玩法，`ArchetypeQuickNav.tsx` 硬编码 count。

**方案**：
- `HotPlaysSection` 改为 Server Component，从 `listPlays()` 取浏览量 Top 5
- `ArchetypeQuickNav` 改为动态计算每个母型下的 Play 数量

### 任务 2：详情页相关推荐（30min）

**问题**：详情页底部没有相关玩法。

**方案**：
- 在 `play/[slug]/page.tsx` 底部添加 "相关玩法" Section
- 推荐逻辑：同母型（基于标签推断）> 同标签 > 随机
- 取 3 条，用 `PlayCard` 或简化卡片展示

### 任务 3：联系表单实际可用（30min）

**问题**：`/contact` 只有静态文本和邮箱链接。

**方案**：
- 添加表单：姓名 / 邮箱 / 主题 / 内容
- 提交到版主后台（写入文件或发送到配置邮箱）
- 最简实现：前端表单 + 调用邮件 API（如用 mailto 回退，或写入本地文件）

### 任务 4：内容占位符清理（批量）

**问题**：部分 `article.mdx` 包含 "这篇是 **MVP 占位**：后续将补充完整内容" 等文本。

**方案**：
- 扫描所有 article.mdx，找出包含占位关键词的文件
- 用 AI 工具批量替换为实际内容（用户说"内容填充后续要使用工具统一替换"）
- 短期：先移除最明显的占位语句，替换为通用的引导文本

## 七、内容填充策略

### 7.1 占位符现状

当前 ~20 篇 Play 中，article.mdx 的质量分层：
- **高质量**（>1000 字符，结构完整）：~10 篇
- **中等**（600-1000 字符，有实质内容）：~6 篇
- **有占位符**（含 "MVP" / "占位" / "TODO"）：~4 篇

### 7.2 处理策略

**短期（本周）**：
- 扫描所有 article.mdx，列出含占位符的文件清单
- 将最明显的占位语句替换为玩法简介（基于 meta.json 的 subtitle + breakdown 自动生成）

**中期（后续用工具）**：
- 用 AI 工具（如当前会话后续调用）批量扩充 article.mdx
- 基于 breakdown 和 codeSnippets 自动生成完整的文章结构
- 每篇目标：>=800 字符，有标题、有小节、有列表

### 7.3 自动文章生成模板

基于已有的结构化数据（breakdown + codeSnippets），可以自动生成：

```markdown
# {title}

{subtitle}

## 玩法概述

（基于 breakdown[0] 生成）

## 核心机制

（基于 breakdown[1+] 生成）

## 关键实现

（基于 codeSnippets 生成）

## 设计要点

（基于 corePoints 生成）
```
