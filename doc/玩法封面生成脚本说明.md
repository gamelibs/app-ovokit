# 玩法封面生成脚本说明

## 脚本位置

`scripts/generate-play-covers.ts`

## 功能

批量为 `content/plays/<slug>/` 下的玩法帖子生成 `cover`（3:4）与 `coverWide`（4:3）封面 SVG，并自动更新 `meta.json`。

- 封面 `cover.svg` 与 `cover-wide.svg` 均统一为 **436×310**（viewBox 0 0 436 310）的 4:3 横版。
- 内容区约 420×294，四周留 8px padding，确保手绘图形在卡片小尺寸下仍清晰可辨。
- 站点卡片、详情页侧边栏、移动端横幅均按 4:3 比例显示，使用 `object-contain` 完整展示。

436×310 在保持 4:3 比例的同时让图形占比更大，配合 `strokeWidth: 3`，在卡片等小尺寸场景下依然清晰可辨。

## 生成方式

脚本支持两种模式：

### 1. 程序化模式（默认 / 推荐批量）

使用本地 `roughjs` 生成统一手绘风格的抽象游戏符号（宝石、方块、塔、骷髅、卡片、灯泡等）。

```bash
pnpm generate:covers
```

等价于：

```bash
pnpm tsx scripts/generate-play-covers.ts --mode=programmatic
```

### 2. AI 模式（基于概念图）

读取 `imgs/compressed/` 中的图片，调用 `/api/ai/generate-cover`，基于截图生成手绘 SVG。需要：

- 本地 Next.js 服务已启动（默认 `http://localhost:13100`，可通过 `NEXT_PUBLIC_SITE_URL` 覆盖）。
- `.env.local` 中已配置 `MOD_PASSWORD`。

```bash
pnpm generate:covers:ai
```

等价于：

```bash
pnpm tsx scripts/generate-play-covers.ts --mode=ai
```

常用参数：

```bash
# 只生成指定玩法
pnpm tsx scripts/generate-play-covers.ts --slug=match-3-retention-and-pacing

# 预览不保存
pnpm tsx scripts/generate-play-covers.ts --dry-run

# 指定输入图片目录（AI 模式）
pnpm tsx scripts/generate-play-covers.ts --mode=ai --input-dir=imgs/compressed

# 调整 AI 转 base64 时的质量
pnpm tsx scripts/generate-play-covers.ts --mode=ai --quality=75
```

## 图标映射规则

程序化模式根据玩法 `tags` 和 `title` 选择图形：

| 关键词 | 图形 |
|--------|------|
| 消除 / match | gem（宝石） |
| 合成 / merge | blocks（方块堆叠） |
| 放置 / idle | sun（太阳） |
| 点击 / clicker / tap | tap（点击手势） |
| roguelike | skull（骷髅） |
| 塔防 / tower defense | tower（塔楼） |
| 跑酷 / run / dodge | runner（跑步小人） |
| 卡牌 / deck / draft | card（扑克卡片） |
| 网格 / grid | grid（网格与移动方块） |
| 随机事件 / dice | dice（骰子） |
| 倒计时 / survival | clock（时钟） |
| 物理 / physics | blocks（方块堆叠） |
| 解谜 / puzzle | puzzle（拼图块） |
| 状态机 / state | flow-process（流程图） |
| 天赋 / talent | lightbulb（灯泡） |
| 抽卡 / gacha | gem（宝石） |
| 其他 | note / star / sparkle / question-mark / lightbulb 轮询 |

脚本中维护了一个 `slugToSketchType` 精确映射表，按每个玩法的主题直接指定最合适的图形，避免纯关键词推断带来的偏差。

为让手绘图形在卡片等小尺寸场景下依然清晰可辨，`flow-process`（流程图处理框）与 `flow-decision`（流程图判断框）采用空心外框 + 内部示意线条的设计，避免大面积 hachure 填充缩小时糊成一团。

## 输出

- SVG 写入 `public/plays/<slug>/cover.svg` 与 `public/plays/<slug>/cover-wide.svg`。
- `content/plays/<slug>/meta.json` 中的 `cover` / `coverWide` 字段会同步更新为对应路径。

## 注意事项

1. 程序化生成的 SVG 使用固定色值（`#faf7ef`、`#202020` 等），与全局设计系统一致；如需动态主题色，可后续改为 CSS 变量。
2. AI 模式受 Kimi API 速率与费用限制，建议少量或指定 slug 使用。
3. 如果 AI 生成失败，脚本会自动降级为程序化生成，不会中断批量流程。
4. 已有占位图 `_placeholders/cover.svg` 仅作为兜底，帖子生成封面后不再引用。
