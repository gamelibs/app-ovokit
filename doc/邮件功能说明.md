# OVO 联系表单通知说明

> 本文档记录联系表单通知功能的完整配置流程、技术方案与上线注意事项。支持 Webhook 实时推送 与 Gmail 邮件发送 两种方式。

---

## 一、功能概述

联系表单（`/contact`）提交后，系统会：
1. 将留言内容保存到本地 `content/contact-messages/` 目录备份。
2. （可选）通过 Webhook 实时推送到 Slack / Discord / 飞书 / 企业微信 / 自研接口。
3. （可选）通过 Gmail API 发送邮件到固定邮箱。

两种通知方式独立工作，至少配置一种即可。

---

## 二、技术原理

```
访客 → /contact → /api/contact
                      ├──→ 保存到 content/contact-messages/（本地备份）
                      ├──→ Webhook 实时推送（可选）
                      └──→ Gmail API 邮件发送（可选）
```

- Webhook：异步、非阻塞，5 秒超时，失败只记录日志，不影响用户提交结果。
- Gmail：使用 `googleapis` 库，用 `refresh_token` 换取短期 `access_token` 后发送邮件。

## 四、Google Cloud 配置

### 1. 创建项目并启用 Gmail API

1. 打开 [Google Cloud Console](https://console.cloud.google.com/)。
2. 创建新项目或选择已有项目。
3. 进入 **APIs & Services** → **Library**。
4. 搜索并启用 **Gmail API**。

### 2. 配置 OAuth 同意屏幕

1. 进入 **APIs & Services** → **OAuth consent screen**。
2. User Type 选择 **External**（外部用户）。
3. 填写应用名称、用户支持邮箱、开发者联系信息。
   - 应用名称示例：`OVO`
   - 用户支持邮箱：`h5gamelog@gmail.com`
4. 保存后进入 **Audience**。
5. 在 **Test users** 中添加：
   ```text
   h5gamelog@gmail.com
   ```

> 应用保持 **Testing** 模式即可长期使用，不需要发布到 Production。只有测试用户能授权。

### 3. 创建 OAuth 2.0 客户端 ID

1. 进入 **APIs & Services** → **Credentials**。
2. 点击 **Create credentials** → **OAuth client ID**。
3. Application type 选择 **Web application**。
4. Name 填写：`OVO Web Client`。
5. Authorized redirect URIs 添加：
   ```text
   http://localhost:13100
   https://developers.google.com/oauthplayground
   ```
   - `localhost:13100` 用于本地授权脚本。
   - `developers.google.com/oauthplayground` 用于 OAuth Playground 获取 token。
6. 点击 **Create**，记下 **Client ID** 和 **Client Secret**。

## 五、获取 Refresh Token

推荐通过 **OAuth Playground** 获取，稳定性最高。

### 使用 OAuth Playground

1. 打开 https://developers.google.com/oauthplayground。
2. 点击右上角 **齿轮图标**（Settings）。
3. 勾选 **“Use your own OAuth credentials”**。
4. 填入：
   - **OAuth Client ID**：上一步获取的 Client ID
   - **OAuth Client secret**：上一步获取的 Client Secret
5. 关闭设置。
6. Step 1 输入 Scope：
   ```text
   https://www.googleapis.com/auth/gmail.send
   ```
7. 点击 **Authorize APIs**。
8. 使用 `h5gamelog@gmail.com` 登录并授权。
9. 点击 **Exchange authorization code for tokens**。
10. 复制 **Refresh token**。

### 使用本地脚本（备用）

如果终端能访问 Google，也可以运行项目脚本：

```bash
pnpm tsx scripts/gmail-auth.ts
```

按提示在浏览器授权并粘贴返回的 code。脚本会输出 refresh token。

## 六、项目环境变量配置

将获取到的凭据填入 `.env.local`：

```bash
# 站点公开配置
NEXT_PUBLIC_SITE_URL=http://localhost:13100
NEXT_PUBLIC_CONTACT_EMAIL=h5gamelog@gmail.com
CONTACT_TO_EMAIL=h5gamelog@gmail.com

# 方式一：Webhook 实时通知（最简单，推荐）
CONTACT_WEBHOOK_URL=https://hooks.slack.com/services/...
CONTACT_WEBHOOK_SECRET=your_webhook_secret_or_token

# 方式二：Gmail API / OAuth 2.0
GOOGLE_CLIENT_ID=你的_Client_ID
GOOGLE_CLIENT_SECRET=你的_Client_Secret
GOOGLE_REFRESH_TOKEN=你的_Refresh_Token
GOOGLE_SENDER_EMAIL=h5gamelog@gmail.com
GOOGLE_REDIRECT_URI=http://localhost:13100

# 算法后端端口（默认 14100，可不配置）
# ALGO_PORT=14100
```

### 线上部署时

将 `NEXT_PUBLIC_SITE_URL` 和 `GOOGLE_REDIRECT_URI` 改为真实域名：

```bash
NEXT_PUBLIC_SITE_URL=https://ovoforge.com
GOOGLE_REDIRECT_URI=https://ovoforge.com
```

同时需要在 Google Cloud Console 的 OAuth 客户端中添加线上域名的 redirect URI。

## 八、常见问题

### 1. OAuth Playground 提示 “This app’s request is invalid”

原因：OAuth Playground 的 redirect URI 没有添加到 Google Cloud Console。

解决：在 **Authorized redirect URIs** 中添加：

```text
https://developers.google.com/oauthplayground
```

### 2. 提示 “Access blocked: app has not completed the Google verification process”

原因：应用处于 Testing 模式，当前登录账号不在 Test users 列表中。

解决：在 OAuth consent screen 的 **Test users** 中添加 `h5gamelog@gmail.com`。

### 3. OAuth Playground GET /users/me/profile 返回 403 insufficient_scope

原因：`gmail.send` 权限只能发邮件，不能读取邮箱资料。

解决：这是正常的。应该测试 `POST /gmail/v1/users/me/messages/send` 发送邮件。

### 4. 网页一直显示“提交中...”

原因：本地 Next.js 服务端无法访问 `gmail.googleapis.com` 或 `oauth2.googleapis.com`，请求挂起。

解决：给终端设置代理后重启服务。例如：

```bash
export https_proxy=http://127.0.0.1:7890
export http_proxy=http://127.0.0.1:7890
./start_web.sh
```

### 5. Access token 倒计时过期怎么办？

OAuth Playground 中显示的 **Access token 倒计时**（如 1913 秒）只是短期 access_token 的有效期，**不影响项目运行**。

项目代码会自动用 `refresh_token` 换取新的 `access_token`，只要 `refresh_token` 不被撤销，邮件功能就能长期工作。

## 九、上线部署注意事项

1. **环境变量迁移**
   - 将本地 `.env.local` 中的 `GOOGLE_CLIENT_ID`、`GOOGLE_CLIENT_SECRET`、`GOOGLE_REFRESH_TOKEN` 配置到线上服务器/Vercel。
   - `GOOGLE_REDIRECT_URI` 和 `NEXT_PUBLIC_SITE_URL` 改为线上域名。

2. **网络连通性**
   - 线上服务器必须能访问 `gmail.googleapis.com` 和 `oauth2.googleapis.com`。
   - 如果服务器在中国大陆，需要确保有海外访问能力（代理、海外服务器等）。

3. **Google Cloud Console 线上配置**
   - 在 OAuth 客户端中添加线上域名的 redirect URI，例如：
     ```text
     https://ovoforge.com
     ```
   - 如果以后需要在线上环境重新获取 refresh_token，这个 redirect URI 是必需的。

4. **Refresh Token 备份**
   - `refresh_token` 一旦丢失，需要重新走 OAuth 授权流程。
   - 建议将 `refresh_token` 安全备份，不要提交到 Git。

5. **应用发布（可选）**
   - 当前应用保持 **Testing** 模式即可。
   - 如果需要让非测试用户也能授权（例如其他管理员邮箱），才需要点击 **Publish App**，并准备隐私政策页面、应用主页等通过 Google 验证。
   - 对于“服务器后台发邮件”这种场景，通常不需要发布应用。

6. **日志与监控**
   - 后端会在控制台输出 `[contact] sending email via Gmail API...` 等日志。
   - 如果发送失败，日志会输出具体错误信息，可用于排查。
