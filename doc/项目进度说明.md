## 1. 项目目的（Why）

### 1.1 核心问题

玩法研发与学习经常遇到三个痛点：

1. 经验难沉淀：玩法拆解与关键代码零散在笔记/聊天记录/仓库片段中，不可检索、不可复用。
2. 学习成本高：缺少"结构化拆解（交互→规则→判定→数据结构→关键算法）"的统一模板。
3. Demo 难共享：演示经常和页面强耦合，导致移动端输入、滚动、样式互相干扰，不利于传播与复用。

### 1.2 目标产出

- 把"玩法"拆成可复用的结构化信息：拆解 / 关键代码 / Demo / 文章（可检索、可复用、可持续迭代）。
- 提供轻量的本地内容发布流程（MVP），让内容可持续增长。
- 为后续更多玩法实例提供底座：嵌入式 Viewer（iframe）+ Demo/API（服务端为执行中心）。

## 2. 站点介绍（What）

### 2.1 一句话介绍

OVOFROGE 是一个面向游戏开发者的"玩法技术实现分享站"，把每个玩法沉淀成结构化内容 + 可嵌入演示，从而支持学习、复用与传播。

### 2.2 目标用户

- 主要：玩法程序/技术策划/客户端工程师（需要快速复现玩法核心交互与判定）
- 次要：新人学习（需要"母型玩法"目录与入门路径）
- 扩展：团队内部知识库（需要可检索与统一结构）

### 2.3 内容形态

- 玩法帖子（Play）：标题/副标题/标签/难度/技术栈/核心点/拆解/代码片段/Demo/文章
- 母型玩法（Archetype）：固定 12 项，作为玩法目录与学习路径的"锚点"
- Demo/Viewer：用于隔离演示交互（建议 iframe 嵌入），避免影响站点页面滚动与样式

### 3.1 V1 目标

一个**手绘风格的、结构化的、可对外公开访问的游戏玩法知识分享站**，具备：
