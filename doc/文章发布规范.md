# OVO 文章发布规范

> 本文档规定如何在 OVO（ovoforge.com）发布玩法文章，包括编辑流程、Markdown 语法约定、内容规范与术语表使用规则。

---

## 一、发布流程

### 1. 进入编辑后台

1. 在网站首页连续点击顶部「OVO」Logo 8 次，打开管理入口。
2. 在侧边栏管理后台输入口令登录。
3. 点击「新建玩法」或进入 `/mod/new`。

### 2. 填写结构化信息

在表单中填写以下字段：

| 字段 | 说明 |
|------|------|
| 标题 | 玩法名称，简洁明确 |
| 副标题 | 一句话概括核心体验 |
| Slug | URL 唯一标识，建议英文/数字/短横线 |
| 标签 | 选择对应的母型、核心原型、玩法特征 |
| 难度 | 简单 / 中等 / 困难 |
| 技术栈 | 实现该玩法使用的主要技术 |
| 核心点 | 3–5 个关键技术/设计点 |
| 封面 | 建议 SVG，也可 PNG/WebP |
| Demo | iframe 嵌入地址或视频 |

### 3. 编写文章

文章使用 Markdown 子集编写，支持标题、加粗、高亮、术语 hover、代码块、列表等。

### 4. 保存草稿

编辑表单会自动保存草稿到浏览器 localStorage。刷新页面后可恢复。

### 5. 发布

点击「发布玩法」后，系统会：

1. 写入 `content/plays/<slug>/meta.json`
2. 写入 `content/plays/<slug>/article.mdx`
3. 生成必要的封面尺寸

## 二、Markdown 语法约定

站点使用自定义轻量 Markdown 渲染器，**不支持**自定义 React 组件。

### 支持的语法

| 语法 | 效果 | 示例 |
|------|------|------|
| `# 标题` | 一级标题（最大） | `# 核心循环分析` |
| `## 标题` | 二级标题 | `## 为什么需要状态机` |
| `### 标题` | 三级标题 | `### 输入延迟` |
| `**加粗**` | 粗体 | `**关键指标**` |
| `==高亮==` | 黄色高亮背景 | `==点击率==` |
| `` `代码` `` | 等宽 inline code | `` `AABB` `` |
| `[术语](glossary:术语)` | 带 hover 释义的术语 | `[状态机](glossary:状态机)` |
| `- 列表项` | 无序列表 | `- 降低输入延迟` |
| `1. 列表项` | 有序列表 | `1. 初始化网格` |
| ` ```js ` | 代码块 | 见下方 |

### 代码块

使用三个反引号包裹代码，可指定语言：

````md
```js
function update() {
  // ...
}
```
````

## 三、术语表使用规则

术语定义集中存放在 `content/glossary.json`。

### 1. 文章中引用术语

当文章中出现专业术语时，使用 glossary 链接：

```md
在动作反应玩法中，[输入延迟](glossary:输入延迟) 是手感的关键因素。
```

渲染效果：术语会带有虚线下划线，鼠标悬停显示释义。

### 2. 添加新术语

如果文章需要使用的术语不在 `content/glossary.json` 中，编辑该文件添加：

```json
{
  "输入延迟": "从玩家操作到游戏画面响应之间的时间差。"
}
```

**规则**：

- 术语名保持简洁，避免过长。
- 释义控制在 50 字以内，适合 tooltip 展示。
- 尽量复用已有术语，避免同一概念多个说法。
- 提交文章时同时提交 glossary.json 修改。

### 3. 术语缺失时的降级

如果 `[术语](glossary:xxx)` 中的术语在 glossary 中不存在，页面会正常显示文字，但不显示 hover tooltip。

## 四、内容规范

### 1. 文章结构建议

一篇标准玩法文章建议包含：

```md
# 标题

## 核心体验

## 关键机制

## 技术实现要点

## 参数与手感

## 常见变体
```

### 2. 文字分层

- **标题**：用 `#`、`##`、`###` 区分层级，标题会自动加粗。
- **关键指标**：用 `==指标名==` 高亮，例如 `==点击率==`、`==留存率==`。
- **重点结论**：用 `**加粗**` 强调。
- **术语**：用 `[术语](glossary:术语)` 添加 hover 释义。

### 3. 配图建议

- 封面：优先使用 SVG，风格保持手绘/线稿。
- 流程图：可用 ASCII 或外部工具生成后上传 SVG/PNG。
- Demo：尽量提供可交互的 iframe 嵌入。

### 4. 禁止事项

- 不要使用 HTML 标签（部分会被过滤或无法渲染）。
- 不要插入外部脚本或 iframe（Demo 字段除外）。
- 不要在文章中暴露管理后台入口或口令相关信息。

---

## 五、注意事项

1. **Slug 唯一性**：发布后修改 Slug 会导致旧链接失效。
2. **草稿备份**：localStorage 草稿仅在当前浏览器有效，提交前请确认内容完整。
3. **术语统一**：同一概念在站内使用同一术语，便于 hover 释义复用。
4. **SEO 友好**：标题和副标题会用于 Open Graph 与搜索引擎摘要。
5. **测试渲染**：发布前可先在本地 `pnpm dev` 预览文章渲染效果。
