# 开发说明（MVP）

本文件用于记录当前已实现功能、核心路由、以及“分层目录”约定，方便后续多人协作与扩展。

相关设计方案见：`doc/母型玩法页面交互设计方案.docx`。

## 当前功能概览

- 内容源：本地文件 `content/plays/<slug>/{meta.json,article.mdx}`
- 母型玩法文案：本地文件 `content/archetypes/<key>/meta.json`
- 核心编辑器原型文案：`content/prototypes/<key>/meta.json`
- 玩法特征文案：`content/features/<key>/meta.json`
- 首页信息流：玩法卡片流、搜索、分组筛选 Tabs、桌面右侧栏（热门标签/新手必读）；移动端 Hero 已折叠为精简横幅
- 母型玩法：`/archetypes` 列表页 + `/archetypes/[key]` 独立详情页（含可玩 Demo、结构解析、常见组合与变体）
- 核心原型：`/patterns` 列表页 + `/patterns/[key]` 独立详情页
- 玩法特征：`/features` 列表页 + `/features/[key]` 独立详情页
- 版主模式：HMAC 签名 Cookie 登录，可在站内发布/编辑玩法（写入本地 `content/plays/<slug>`）
- 算法后端（Demo）：`server`（Fastify + Zod），提供 Demo API 与三消等试玩逻辑
- 图片优化：核心路径封面图已迁移到 `next/image`，支持响应式 `srcset` 与 SVG `unoptimized`
- 联系通知：联系表单除本地落盘与可选 Gmail 外，新增通用 Webhook 实时推送
- 搜索体验：搜索索引已扩展至 breakdown、codeSnippets、demo.note、article.mdx 正文，支持热门搜索建议与结果高亮

## 路由与入口

### 页面路由（Next.js App Router）

本项目使用 **Route Group** 做职责隔离：

- `(site)`：带站点壳（TopNav/BottomNav）的常规页面
- `(embed)`：iframe/嵌入专用页面（不挂站点壳，避免样式/布局/最小宽度干扰）

主要路由：

- 首页：`/` → `src/app/(site)/page.tsx`
- 玩法详情：`/play/[slug]` → `src/app/(site)/play/[slug]/page.tsx`
- 母型玩法列表：`/archetypes` → `src/app/(site)/archetypes/page.tsx`
- 母型玩法详情：`/archetypes/[key]` → `src/app/(site)/archetypes/[key]/page.tsx`
- 核心原型列表：`/prototypes` → `src/app/(site)/prototypes/page.tsx`
- 核心原型详情：`/prototypes/[key]` → `src/app/(site)/prototypes/[key]/page.tsx`
- 玩法特征列表：`/features` → `src/app/(site)/features/page.tsx`
- 玩法特征详情：`/features/[key]` → `src/app/(site)/features/[key]/page.tsx`
- 版主页面：`/mod/*` → `src/app/(site)/mod/*`
- BlockKit 编辑器（创建工具）：`/demo/blocks` → `src/app/(site)/demo/blocks/page.tsx`
- BlockKit Viewer（展示/可嵌入）：`/embed/blocks/[templateId]` → `src/app/(embed)/embed/blocks/[templateId]/page.tsx`
- 三消 Demo 开发页：`/demo/match3` → `src/app/(site)/demo/match3/page.tsx`
- 三消 Viewer（展示/可嵌入）：`/embed/demos/match3` → `src/app/(embed)/embed/demos/match3/page.tsx`
- 母型 Demo 嵌入：`/embed/archetypes/[key]` → `src/app/(embed)/embed/archetypes/[key]/page.tsx`

### 首页筛选参数（URL）

首页非默认落地（信息流页）使用 query 参数驱动：

- `all=1`：进入信息流筛选页（否则为默认落地 Hero）
- `group`：顶部分组（`archetype` / `feature` / `difficulty`）
- `cat`：分组下的具体分类 key（如 `match-clear`、`merge`、`beginner` 等）
- `q`：全文搜索（匹配标题/副标题/tags/techStack/corePoints/breakdown/codeSnippets/demo.note/article.mdx 正文）

说明：

- 点击顶部分组 Tab：`母型玩法` 会跳转到 `/archetypes`；`玩法特征/难度层级` 保持在 `/` 并带上 `?group=...&all=1`。
- 分类与分组解析在 `src/lib/content/plays.ts` 的 `resolvePlayBrowseState()` 中，兼容旧 `cat` key 的映射（避免旧链接失效）。

## 母型玩法（12 个固定项）

母型玩法列表为固定 12 个（稳定、不可随意变）：

1. 消除（Match / Clear）
2. 躲避（Dodge / Avoid）
3. 行进 / 跑酷（Runner）
4. 射击（Shoot / Aim）
5. 战斗对抗（Combat）
6. 放置 / 建造（Placement）
7. 策略决策（Choice / Strategy）

9. 解谜（Puzzle）

对应数据定义：`src/lib/archetypes/archetypes.ts`（key、文案、默认规则提示、学习目标占位）。

## 分层目录约定（建议）

为了区分“数据/业务/展示/路由”，新增与维护功能时建议按以下层级组织：

### 1) `src/lib/*`（数据与基础能力）

纯数据与基础能力（类型、常量、读取/解析、工具函数），不直接依赖 React 组件。

- 玩法内容模型与读取：`src/lib/content/plays.ts`
- 母型玩法定义：`src/lib/archetypes/archetypes.ts`
- 核心编辑器原型定义：`src/lib/prototypes/prototypes.ts`
- 玩法特征定义：`src/lib/features/features.ts`
- 母型玩法文案读取：`src/lib/archetypes/spec.ts`
- 母型玩法图片读取：`src/lib/archetypes/assets.ts`
- 原型文案读取：`src/lib/prototypes/spec.ts`
- 特征文案读取：`src/lib/features/spec.ts`

### 2) `src/features/*`（业务与页面模型）

将“数据”转换为页面可消费的“页面模型”（Page Model），聚合占位/默认值、组合逻辑等；避免把业务规则塞进 UI 组件。

- 母型玩法页面模型：`src/features/archetypes/pageModel.ts`

### 3) `src/components/*`（UI 组件）

只关心展示与交互（props 驱动），尽量不包含业务决策。

- 首页顶部筛选：`src/components/plays/BrowseGroupTabs.tsx`、`src/components/plays/CategoryTabs.tsx`
- 母型玩法页面结构：`src/components/archetypes/ArchetypePage.tsx`

### 4) `src/app/*`（路由与组装）

负责路由参数解析、加载页面模型、组装组件。

- 母型玩法路由：`src/app/(site)/archetypes/page.tsx`、`src/app/(site)/archetypes/[key]/page.tsx`
- 首页路由：`src/app/(site)/page.tsx`

## BlockKit（创建工具 vs 展示 Viewer）

目标：把“产出交互案例的数据（创建）”与“加载数据进行演示（展示）”分离，避免演示时的输入/滚动/布局影响站点页面，并为后续扩展更多玩法实例做底座。

### 模块职责

- 创建工具（Editor）
  - 入口：`src/features/block-editor/*` + 页面 `src/app/(site)/demo/blocks/page.tsx`
  - 能力：平移/缩放、多选框选、网格吸附、撤销重做、属性面板、JSON 导入导出（用于“产出数据”）
- 展示 Viewer（只负责加载数据并演示）
  - 组件：`src/components/block-kit/BlockViewer.tsx`
  - 嵌入路由：`src/app/(embed)/embed/blocks/[templateId]/page.tsx`（用于 iframe）
  - 站点使用：首页演示 `src/components/home/InteractiveHero.tsx` 通过 iframe 嵌入，展示与页面交互隔离

### 模板数据（Viewer 的数据源）

当前 Viewer 使用模板 ID 加载数据（后续可扩展为从内容文件/服务端读取 JSON）：

- 模板定义：`src/lib/block-kit/templates.ts`
- 新增玩法实例：在 `blockTemplates` 中追加一个条目，并为其分配唯一 `id`（对应 `/embed/blocks/<id>`）

### iframe 使用建议

- 推荐在站内展示时使用 iframe：隔离滚动/手势/样式，减少“演示组件影响页面交互”的问题。
- iframe 容器建议使用固定比例（如 `aspect-video` 或本项目的 `aspect-[12/7]`）+ `w-full`，移动端自适配。
- 嵌入地址：`/embed/blocks/<templateId>`（例：`/embed/blocks/drag-pairs`）

### 移动端输入/滚动隔离（关键改动）

为避免移动端拖拽时触发页面滚动/回弹/手势竞争：

- `src/components/block-kit/BlockCanvas.tsx` 使用 `setPointerCapture(pointerId)`，后续 `pointermove/up/cancel` 仅在 canvas 上处理（不再依赖 `window` 级监听）
- 默认 `touchAction: none`，并在触摸 pointer 事件中 `preventDefault()`（在 iframe/嵌入场景下更稳定）

## 母型玩法图片规范（占位图策略）

母型玩法详情页建议使用 **3 张核心说明图 + 1 张可选扩展图**（超过 4 张会更像美术展示而不是教学）。

| 图序 | 作用 | 是否必需 |
| --- | --- | --- |
| 图 1 | 玩法母型概览图（Hero） | ✅ 必须 |
| 图 2 | 核心交互说明图（Interaction） | ✅ 必须 |
| 图 3 | 规则 / 判定结构图（Rule） | ✅ 必须 |
| 图 4 | 变体 / 组合示例图（Advanced） | ⭕ 可选（建议折叠区） |

页面展示比例建议：

- 图 1（Hero）：桌面端 100% 宽、约 `~420px` 高；移动端 100% 宽、约 `~260px` 高
- 图 2（Interaction）：桌面端约 70% 宽；移动端 100% 宽
- 图 3（Rule）：桌面端约 70% 宽；移动端 100% 宽
- 图 4（Advanced）：桌面端约 50–60% 宽；移动端折叠展示

对应占位实现：`src/components/archetypes/ArchetypePage.tsx`。

## 试玩 Demo（服务端为执行中心）

原则：Demo 的执行逻辑在 `server` 端；前端只负责输入/输出与渲染。

### Demo API

- `GET /api/demos`：Demo 列表（meta）
- `GET /api/demos/:id`：Demo 元信息（meta）
- `POST /api/demos/:id/init`：初始化（输入为 initSchema）
- `POST /api/demos/:id/step`：推进一步（输入 `{ state, action }`，action 走 actionSchema）

实现位置：

- Demo registry：`server/demos/registry.ts`
- Demo types：`server/demos/types.ts`
- 示例模块（三消）：`server/demos/match3/*`

说明：

- 站内 iframe/Viewer 默认通过 Next.js Route Handler 提供同路径的 API（`src/app/api/demos/*`），这样前端可用相对路径调用，无需额外域名与 CORS 配置。

### 示例：三消（match3）

已实现可玩 Demo：

- `id`: `match3`
- `init`：生成无初始三连的棋盘（seeded），返回 `{ state, view }`
- `step`：支持相邻交换 + 成消判定（不成消则回退）+ 消除/下落/补齐 + 连锁结算（events 驱动表现）
- 前端 Viewer：`src/components/demos/DemoEmbed.tsx`、`FullscreenStage.tsx` 等组件提供工具栏、全屏、重开能力

此外，母型详情页 `/archetypes/[key]` 会读取 `archetypeDemos` 配置，将匹配的核心原型或具体 Demo 嵌入到页面中。
