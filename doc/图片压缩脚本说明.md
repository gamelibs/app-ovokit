# 图片压缩脚本说明

> 将任意目录下的大体积图片批量压缩为站点可用的 WebP 格式。

---

## 一、脚本位置

```
scripts/compress-images.ts
```

依赖：`sharp`（已安装到 `devDependencies`）。

---

## 二、默认行为

- **输出格式**：WebP
- **最大尺寸**：1200×1200（等比缩放，不放大）
- **WebP 质量**：80
- **不覆盖原图**：原文件保留，压缩结果放在新目录

> 注意：脚本不再内置 `imgs/` 默认目录，必须显式指定 `--input-dir` 与 `--output-dir`。

---

## 三、使用方法

### 1. 查看帮助

```bash
pnpm compress:images
```

### 2. 压缩到指定目录

```bash
pnpm tsx scripts/compress-images.ts --input-dir=./raw --output-dir=./public/imgs
```

### 3. 自定义参数

```bash
# 更低质量、更小体积
pnpm tsx scripts/compress-images.ts --input-dir=./raw --output-dir=./public/imgs --quality=70 --max-width=1024

# 原地替换为 WebP（谨慎使用）
pnpm tsx scripts/compress-images.ts --input-dir=./raw --in-place
```

| 参数 | 说明 | 默认值 |
|------|------|--------|
| `--input-dir` | 输入图片目录 | **必填** |
| `--output-dir` | 输出目录 | **必填**（`--in-place` 时除外） |
| `--quality` | WebP 质量（1-100） | `80` |
| `--max-width` | 最大宽度 | `1200` |
| `--max-height` | 最大高度 | `1200` |
| `--in-place` | 直接替换原文件为 WebP | 否 |

## 四、压缩效果示例

对 36 张 PNG 素材执行默认压缩：

```
原始大小：53.67 MB
压缩后：  1.88 MB
整体节省：96.5%
```

单张从约 1.5MB 降至 40–80KB，视觉质量在网页展示场景下基本无损。

---

## 五、在站点中使用

压缩后的图片位于你指定的输出目录，直接复制到 `public/` 下对应目录即可在站点引用。

例如：

```bash
mkdir -p public/imgs
rsync -av ./public/imgs-src/ public/imgs/
```

页面中引用：

```tsx
<Image src="/imgs/minigame_concepts_3x4_00001.webp" alt="" width={900} height={1200} />
```

---

## 六、注意事项

1. **原图备份**：默认不覆盖原图，建议保留原图作为高清源。
2. **透明 PNG**：WebP 支持透明通道，无需担心。
3. **动画 GIF**：脚本会处理 GIF，但 `sharp` 对动画支持有限，只会保留第一帧；有动画需求的 GIF 请单独处理。
4. **质量选择**：
   - `80` 适合大多数网页展示场景
   - `70` 适合缩略图/列表页
   - `90` 适合需要高保真的封面/详情图
5. **批量复制**：如需把压缩图同步到 `public/`，可写一个简单的 `cp`/`rsync` 命令，或扩展脚本自动复制。
